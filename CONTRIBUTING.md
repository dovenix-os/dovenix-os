# Contributing to Dovenix

Dovenix is in the design phase — the most valuable contributions right now are design
review and docs. This file mostly exists to pin down the licensing rules before the
first line of code lands, because they cannot be fixed retroactively.

## Licensing policy

### The default: `Apache-2.0 OR MIT`

All code and documentation in this repository is dual-licensed under Apache-2.0 OR MIT
(the Rust ecosystem convention), **except** `drivers/ported/` (see below). By
submitting a contribution you agree to license it under these terms.

Every source file must carry an SPDX header:

```rust
// SPDX-License-Identifier: Apache-2.0 OR MIT
```

### Why dual and not Apache-2.0 alone

Linux is GPL-2.0-only, and Apache-2.0 is **not** GPLv2-compatible. Drivers ported from
Linux must link against our driver SDK, IPC stubs, and wire-protocol codecs. Dual
licensing every library lets a GPL-2.0-only driver link them under MIT while everyone
else uses Apache-2.0 (with its patent grant). This is why the dual license is
mandatory for anything a driver might link — which in practice means everything under
`libs/`.

### The exception: GPL islands (`drivers/ported/`)

Drivers ported from Linux source are derivative works and are licensed
**GPL-2.0-only**, with SPDX header:

```rust
// SPDX-License-Identifier: GPL-2.0-only
```

Rules for GPL islands:

1. GPL-derived code lives **only** under `drivers/ported/<driver-name>/`, one
   directory per driver, each with its own `LICENSE` file and provenance note
   (upstream source, kernel version, commit).
2. A GPL island is always a **separate process** communicating with the rest of the
   system exclusively via the driver wire protocol. Never a library.
3. Code outside `drivers/ported/` must **never** copy from, or be written while
   referencing, GPL source. This includes the Linux-API compatibility shim
   (`libs/linux-shim/` when it exists), which must be written clean from documentation
   and driver-facing API surfaces, and is itself `Apache-2.0 OR MIT`.
4. If you are unsure whether something crosses the line, ask in the issue tracker
   *before* writing the code.

### Third-party code

Permissively licensed code (MIT, BSD-2/3, ISC, Apache-2.0) may be vendored with its
license and attribution preserved. No GPL/LGPL/AGPL/MPL code outside
`drivers/ported/`. No code of unknown provenance, and no code generated by tools
trained to reproduce GPL sources verbatim.

## Development principles

- **Rust first.** Kernel and system components are Rust (`no_std` where required).
  `unsafe` is confined to narrow, documented modules.
- **Tests are end-to-end or integration, never unit.** A component is tested by
  booting a partial system (QEMU) with a test-harness component speaking real
  protocols to it.
- **Protocols are contracts.** Any change to a wire protocol requires a version bump
  and a design-doc update in the same change.
- **Monorepo, modular.** Each top-level directory must remain independently buildable
  and extractable — no reaching across component boundaries except through `libs/`.
